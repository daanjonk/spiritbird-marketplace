# Weekly Review Template

Use this structure for weekly reviews saved to `Analytics/weekly-review-YYYY-MM-DD.md`.

```markdown
---
type: weekly-review
date: YYYY-MM-DD
week: [week number]
---

## Numbers
- **Revenue:** [amount or trend]
- **Traffic:** [if known]
- **Conversions:** [if known]
- **Best performer:** [product, campaign, or channel]
- **Biggest problem:** [what underperformed]

## What Worked
[Their answer — what felt good this week]

## What Didn't
[Their answer — what was hard or broken]

## Last Week's Commitments
- [x] or [ ] [commitment from previous review]

## Next Week's Priorities
1. [priority 1]
2. [priority 2]
3. [priority 3]

## One Thing I Learned
[Their insight — the most valuable part of the review]
```

### Notes for Claude:
- File goes in `Analytics/` folder (Growth+ tier only for standalone file)
- For Foundation tier, append a simplified version to the daily note instead
- Always check the previous weekly review to pull in last week's commitments
- If the user doesn't track exact numbers, "up/down/flat" is fine — don't push
- The "One Thing I Learned" is the gold — if it reveals customer/market/competitor insight, route a copy to the relevant vault file
- Keep the tone reflective, not bureaucratic
