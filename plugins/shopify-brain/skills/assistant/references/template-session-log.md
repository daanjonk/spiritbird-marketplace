# Session Log Template

Append this to the current `Daily/YYYY-MM-DD.md` when meaningful work happens during a session.

```markdown
## Session: [HH:MM] — [2-5 word topic]
**What we worked on:** [1-2 sentences]
**Key outcomes:** [decisions made, content created, insights captured]
**Files updated:** [list of vault files that changed this session]
**Next steps:** [anything to follow up on]
```

### Notes for Claude:
- Only log sessions where real work happened — not casual chat
- Keep it scannable: short lines, no paragraphs
- Multiple sessions in one day = multiple session blocks, appended chronologically
- "Files updated" helps the user see their vault growing — always include it
- If a decision was made, also log it in `Decisions/decision-log.md` — the session log is just a pointer
- Time can be approximate — "afternoon" is fine if exact time isn't known
